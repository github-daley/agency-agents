# 例（Examples）

このディレクトリには、エージェンシーのエージェント群を協調させて実世界のタスクに対処する例出力が含まれます。

## 目的

agency-agents リポジトリはエンジニアリング、デザイン、マーケティング、プロダクト、サポート、空間コンピューティング、プロジェクト管理など多数の専門エージェントを定義しています。しかし、エージェント定義だけでは「これらを同時に運用したときに何が起きるか」は示せません。

これらの例は「フルエージェンシーが協力すると実際にどうなるか？」という問いに答えます。

## コンテンツ

### [nexus-spatial-discovery.md](./nexus-spatial-discovery.md)

**内容:** 8 つのエージェントが並列に動いて、ソフトウェア機会を評価し、統合された計画を作成した完全なプロダクト発見演習。

**シナリオ:** ウェブ調査で AI エージェントオーケストレーションと空間コンピューティングの交差点に機会が見つかり、エージェンシー全体を同時に動員して市場検証、技術アーキテクチャ、ブランド戦略、GTM、サポート設計、UX リサーチ、35 週間の実行計画などを生成しました。

**使用エージェント:** Product Trend Researcher, Backend Architect, Brand Guardian, Growth Hacker, Support Responder, UX Researcher, Project Shepherd, XR Interface Architect

**重要なポイント:** 8 つのエージェントが並列に実行され、協調コストを最小化しつつ一貫した成果が得られました。

## 新しい例の追加方法

興味深いマルチエージェント演習を実行したら、ここに追加してください。良い例は次を示します：

- 複数のエージェントが共有目標で協力していること
- エージェンシーの能力の幅が表現されていること
- 実運用に近い適用例であること
