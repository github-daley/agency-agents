# MCP Memory 統合

> MCP（Model Context Protocol）を使って、任意のエージェントにセッション間の永続メモリを持たせます。

## 概要

デフォルトではエージェントはセッションごとに初期化され、コンテキストは手動でコピーして渡します。MCPメモリサーバーを導入すると：

- **セッション間メモリ**: エージェントが過去の決定や成果物、文脈を記憶します
- **ハンドオフの連続性**: 別のエージェントに受け渡す際、受け取る側が前回の作業を正確に参照できます
- **ロールバック**: QAで失敗した場合、既知の良好な状態に戻せます

## セットアップ

MCPの `remember`, `recall`, `rollback`, `search` ツールを提供するサーバーを MCP クライアント設定に追加します：

```json
{
  "mcpServers": {
    "memory": {
      "command": "your-mcp-memory-server",
      "args": []
    }
  }
}
```

互換する任意のMCPサーバーを使用できます。実装一覧は [Model Context Protocol](https://modelcontextprotocol.io) を参照してください。

## エージェントへメモリを追加する方法

既存エージェントに永続メモリを付与するには、エージェントのプロンプトに **Memory Integration** セクションを追加します。エージェントは重要なタイミングでMCPツールを使うよう指示されます。

### パターン

```markdown
## Memory Integration

セッション開始時:
- 過去セッションの関連コンテキストを検索して呼び出す（エージェント名とプロジェクト名をタグとして使用）
- エージェント名でタグ付けされたメモリを確認して、作業の継続地点を決める

重要な決定や成果が出たとき:
- 決定や成果をタグ付きで `remember` する（エージェント名、プロジェクト名、トピックなど）
- 将来のセッションや別エージェントが理解できるよう、理由を含めて記録する

ハンドオフ時:
- 次のエージェント向けのタグ付きメモリを記録する（完了事項、保留、次に必要な情報）

失敗時の回復:
- 最後の正常状態を検索し、`rollback` で復元する
```

## MCPツールが行うこと

- `remember` — 決定やスナップショットをタグ付きで保存
- `recall` — キーワードやタグ、意味的類似性で関連メモを検索
- `rollback` — 前の状態に戻す
- `search` — セッション横断でメモを検索

コード変更は不要で、エージェントファイルの編集のみで機能します。

## メモ

- 一貫したタグ付け（エージェント名とプロジェクト名）を推奨します
- LLMは指示に従って重要性を判断します
- ロールバックは回復の要 — Reality Checker が失敗した場合に強力な手段となります
